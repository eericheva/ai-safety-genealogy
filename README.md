# AI Safety / Alignment Genealogy (2005–2026)

Data + reproducible visualisations for a long-read on how the field of AI safety was born, funded, and re-focused over time — how research directions ("tracks") emerged, expanded, peaked, and were absorbed or faded, and how money and scientific attention flowed between organisations and tracks.

<https://mrswallbreakermaps.com/genealogy/index.html>

## Core principle — everything is data-first

Every chart is generated by `build_viz.py` from `data/events.csv` (the single source of truth). Every row in `events.csv` carries a source URL and a confidence level. Claims that could not be verified against a primary source are explicitly flagged in the data (inline `[flag ...]` notes + a `confidence` column), not silently kept.

## Repo layout

```
data/
  events.csv        # THE backbone: one atomic fact per row (year, actor, track, event_type, $, source_url, confidence)
  actors.csv        # registry of organisations/funders/programs (type, first_seen, status)
  taxonomy.md       # the track ontology (directions) — big/small/emerging
  sources.csv       # registry of the 63 key structured sources (S1-S63) + status
  pubcounts.csv     # arXiv keyword paper-counts per track per year (proxy for scientific attention) — 23 proxy directions + the dedup `_safety_corpus` line
  verification.csv  # per-source verification tracker (88 rows)
build_viz.py        # reads data/*.csv -> writes viz/*.html + viz/img/*.png (Plotly). Rerun after any data edit.
collect_*.py        # source collectors (arXiv counts, LTFF, SFF) that feed data/
report.md           # auto-generated summary tables + coverage/total control
viz/                # generated charts: interactive *.html + static img/*.png + index.html gallery
POST.md             # the long-read post itself
```

## events.csv schema

`year, date, actor, actor_type, direction, event_type, scale, detail, source_url, source_type, confidence`

- `event_type`: founded, closed, pivot, grant_out, grant_in, publication, program_launch, program_end, statement, hire_lead, **funding_total** (aggregate money-by-track-by-year; excluded from the illustrative Sankey to avoid double-counting).
- `direction`: track (see `taxonomy.md`). `mixed_technical` = field-level aggregates.
- `scale`: USD amount where applicable.
- `confidence`: high / med. Rows with unverifiable specifics carry an inline `[flag ...]` note.

## Caveats (must be stated in the post)

1. arXiv keyword counts ≠ bibliometrics (term lags the real track start).
2. "events per track" in `events.csv` reflects collection density, not real-world activity — use money + publications for activity, not event counts.
3. Causal "why" claims are interpretation over verified events, not sourced facts.
